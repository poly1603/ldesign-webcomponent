# 组件文档编写规范

本文档定义了 LDesign WebComponent 组件库文档的统一编写标准。

## 文档结构模板

每个组件文档应包含以下标准章节：

```markdown
# ComponentName 组件名称

简洁的一句话描述组件用途。

## 何时使用

- 使用场景 1
- 使用场景 2
- 使用场景 3

## 代码演示

### 基础用法

描述最简单的用法。

<div class="demo-container">
  <!-- 实际可运行的示例 -->
</div>

```html
<!-- 代码示例 -->
```

### 高级功能 1

描述高级功能。

<div class="demo-container">
  <!-- 示例 -->
</div>

```html
<!-- 代码 -->
```

## 框架集成

### Vue 3 使用

```vue
<script setup lang="ts">
// Vue 3 示例代码
</script>

<template>
  <!-- 模板代码 -->
</template>
```

### React 使用

```tsx
import React from 'react';

function Example() {
  // React 示例代码
  return (
    // JSX 代码
  );
}
```

| 事件名 | 说明 | 回调参数 |

## 编写要求

### 1. 标题和描述

- **组件名称**：中英文，格式为 `# ComponentName 组件名称`
- **一句话描述**：简洁明了，说明组件的核心用途
- **何时使用**：3-5个具体的使用场景，使用列表格式

### 2. 代码演示

每个示例必须包含：

1. **清晰的标题**：使用 `###` 三级标题
2. **功能描述**：一段文字说明该示例展示的功能
3. **实际演示**：
   ```html
   <div class="demo-container">
     <!-- 实际可运行的组件代码 -->
   </div>
   ```
4. **代码展示**：
   ````markdown
   ```html
   <!-- 完整的可复制代码 -->
   ```
   ````

### 3. API 文档

#### Props 表格要求

| 列名 | 要求 |
|------|------|
| 属性 | 准确的属性名，使用代码格式 |
| 说明 | 详细描述属性用途和影响 |
| 类型 | 使用 TypeScript 类型标注，用反引号包裹 |
| 默认值 | 明确标注默认值，用反引号包裹 |
| 版本 | 标注新增版本，如 `2.0.0`，可选 |

**示例：**

```markdown
| 属性 | 说明 | 类型 | 默认值 | 版本 |
|------|------|------|--------|------|
| `type` | 按钮类型，影响按钮的视觉样式和语义 | `'primary' \| 'default' \| 'dashed'` | `'default'` | - |
| `size` | 按钮尺寸 | `'small' \| 'medium' \| 'large'` | `'medium'` | - |
| `disabled` | 是否禁用按钮，禁用后无法点击 | `boolean` | `false` | - |
```

#### Events 表格要求

必须包含：
- 事件名（原生事件名）
- 详细说明（何时触发）
- 完整的回调参数类型定义

#### Methods 表格要求

必须包含：
- 方法名
- 用途说明
- 参数类型
- 返回值类型

### 4. 框架集成示例

**必须提供的框架示例：**
- Vue 3（使用 Composition API）
- React（使用 Hooks）

**可选提供的框架示例：**
- Angular
- Svelte
- 原生 HTML/JavaScript

每个示例必须：
- 使用实际可运行的完整代码
- 包含必要的导入语句
- 展示常见的使用模式
- 代码风格符合框架最佳实践

### 5. 样式自定义

如果组件支持 CSS 变量自定义，必须提供：
- 完整的 CSS 变量列表
- 每个变量的默认值
- 变量的作用说明
- 自定义示例

### 6. 最佳实践

提供明确的推荐和不推荐做法：

```markdown
### 最佳实践

✅ **推荐做法**
- 使用语义化的类型属性，如 `type="primary"` 表示主要操作
- 为重要操作使用 `loading` 状态提供视觉反馈

❌ **不推荐做法**  
- 在同一区域使用多个 primary 按钮，造成用户选择困难
- 禁用按钮后不提供禁用原因的提示
```

## 示例质量标准

### 优秀示例的特征

✅ **代码清晰**
- 代码格式规范，缩进统一
- 变量命名语义化
- 包含必要的注释

✅ **功能完整**
- 示例可直接运行
- 展示了核心功能
- 边界情况有处理

✅ **说明充分**
- 每个示例都有文字说明
- 关键配置有注释解释
- 提供了使用场景描述

### 避免的问题

❌ **示例不完整**
- 缺少必要的导入语句
- 代码无法直接运行
- 依赖未声明的变量

❌ **说明不足**
- 没有文字说明示例用途
- 代码缺少关键注释
- 未说明示例的使用场景

- [ ] API 表格完整，所有属性都有说明
- [ ] 提供了 Vue 3 和 React 集成示例
- [ ] 代码示例可以直接运行
- [ ] 包含了最佳实践说明
- [ ] 检查了无障碍性说明
- [ ] 列出了相关组件
